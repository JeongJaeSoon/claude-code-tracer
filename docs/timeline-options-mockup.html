옵션 A: 헤더 통합형 (Integrated Header Bar)

높이: 24px
⚡
2026-01-09 6d186dc8-e141-4ee7
Run Feedback Metadata

헤더 바로 아래에 얇은 도트 라인으로 타임라인을 표시합니다. 각 점은 하나의 Turn(사용자 메시지)을 나타냅니다.

  • 각 점(●)이 하나의 Turn
  • 점의 색상은 해당 Turn의 주요 도구 색상
  • 선택된 Turn은 강조 표시
  • 호버 시 확대 + 글로우 효과
장점

최소 공간 사용, 전체 흐름 파악 용이, 헤더와 자연스럽게 통합

단점

도구 수, 실행 시간 등 세부 정보 표현 제한

옵션 B: 미니 프로그레스 바 (Mini Progress Timeline)

높이: 32px

단일 프로그레스 바 형태로 각 Turn을 색상 블록으로 표시합니다. 시간 비율이 직관적으로 표현됩니다.

  • 색상 블록으로 Turn 표시 (도구별 색상)
  • 블록 너비 = 해당 Turn의 실행 시간 비율
  • 빈 시간은 투명 처리
  • 클릭으로 해당 Turn 선택
장점

시간 비율 직관적, 도구 분포 한눈에 파악, 연속적인 흐름 표현

단점

Sub-agent 구분 어려움, 짧은 Turn 클릭 어려움

옵션 C: 스파크라인 + 도트 (Sparkline with Dots)

높이: 48px

각 도트가 하나의 Turn을 나타내며, 도트의 크기로 도구 호출 수를 표현합니다.

  • 각 도트 = 하나의 Turn (role: "user")
  • 도트 크기 = 도구 호출 수 (많을수록 큼)
  • 도트 색상 = 주요 도구 색상
  • 연결선으로 시간 흐름 표현
  • 선택된 도트: 링 하이라이트
장점

최소 공간으로 최대 정보, Turn 수와 분포 직관적, 클릭/호버 인터랙션 명확

단점

옵션 A보다 높음, 많은 Turn에서 도트 겹침 가능

옵션 D: TreeView 내 인라인 (Inline in TreeView)

Sidebar 통합

TreeView 헤더에 미니 타임라인을 통합하고, 각 트리 노드와 타임라인이 동기화됩니다.

  • TreeView 헤더에 미니 타임라인 통합
  • 타임라인 클릭 → TreeView 스크롤 동기화
  • 각 Turn 노드 옆에 해당 위치의 컬러 도트
  • 메인 영역에서 별도 타임라인 공간 불필요
장점

메인 영역 공간 100% 활용, TreeView와 자연스러운 연결, 직관적인 네비게이션

단점

사이드바 접으면 타임라인도 숨김, 세로 공간 활용
